## 📌 Overview

Studying from 500-page textbooks is exhausting. Finding specific topics for exams is time-consuming. **University Notes Assistant** solves this.

Upload your lecture notes or academic PDFs, ask questions in plain English, and receive precise, AI-generated answers — all grounded in your own documents with source references included.

> **Built by a CS/AI student, for CS/AI students.**

## 🧠 How It Works

This project is built on **RAG (Retrieval-Augmented Generation)** — a modern AI architecture that combines the power of vector search with large language models.

```
📄 PDF Upload
     │
     ▼
📦 Text Chunking (LangChain)
     │
     ▼
🔢 Embedding Generation (OpenAI Embeddings)
     │
     ▼
🗄️ Vector Storage (ChromaDB)
     │
     ▼
❓ User Question
     │
     ▼
🔍 Semantic Retrieval (Top-K similar chunks)
     │
     ▼
🤖 LLM Generation (GPT-4 + Retrieved Context)
     │
     ▼
💬 Answer + Source References
```

### Step-by-Step Flow

1. **Upload** — User uploads a PDF, saved to `data/uploads/`
2. **Ingest** — PDF is read, split into chunks, embedded, and stored in ChromaDB (`data/chroma_db/`)
3. **Query** — User asks a question; the system retrieves the most semantically similar chunks
4. **Generate** — Retrieved chunks + question are sent to the LLM, which returns a grounded answer
5. **Display** — The frontend shows the answer alongside source references (page + excerpt)

## 🚀 Quick Start

### Prerequisites

- Python 3.11+
- An [OpenAI API Key](https://platform.openai.com/api-keys)
- Docker (optional, for containerized run)

---

### Option 1: Run Locally

**1. Clone the repository**
```bash
git clone https://github.com/your-username/university-notes-assistant.git
cd university-notes-assistant
```

**2. Set up environment variables**
```bash
cp .env.example .env
```
Open `.env` and add your OpenAI API key:
```env
OPENAI_API_KEY=your_openai_api_key_here
```

**3. Install dependencies**
```bash
uv sync
```

**4. Run the app**
```bash
uv run uvicorn app.main:app --reload
```

**5. Open in browser**
```
http://127.0.0.1:8000
```

---

### Option 2: Run with Docker 🐳

```bash
docker compose up --build
```

Then open:
```
http://localhost:8000
```
